# ZooCute UI 重设计规格

**日期：** 2026-03-28
**状态：** 已批准

---

## 目标

将现有的单一布局（topbar + 三栏玻璃态）重构为 IDE 风格的 Ribbon 导航布局，参考 tiny-rdm 和 PrettyZoo 的交互模式，提升信息密度和操作效率。

---

## 整体布局

应用由三个区域组成，从左到右：

```
┌──────┬────────────────────┬─────────────────────────────────┐
│      │                    │  Stat 元数据网格                 │
│Ribbon│  左面板             ├─────────────────────────────────┤
│      │  (树 / 连接列表)    │  编辑器工具栏                    │
│ 48px │  可拖拽宽度         │  编辑器正文                      │
│      │  默认 220px         │                                  │
└──────┴────────────────────┴─────────────────────────────────┘
```

- **Ribbon**：固定 48px 宽，竖排图标导航，切换左面板内容模式
- **左面板**：宽度可拖拽调整，最小 160px，最大 400px，宽度持久化到 `localStorage`
- **右侧内容区**：占剩余宽度，上方 Stat 网格 + 下方编辑器

---

## Ribbon

固定在最左侧，包含以下入口（从上到下）：

| 图标 | 模式 | 说明 |
|------|------|------|
| 🌲 | 树浏览（Browse） | 默认模式，左面板显示 ZK 节点树 |
| 🔌 | 连接管理（Connections） | 左面板显示已保存连接列表 |
| 📋 | 操作日志（Log） | 占位，后续扩展（当前不实现内容） |
| ⚙️ | 设置（底部对齐） | 占位，后续扩展 |

当前激活模式的图标有高亮背景（深色：`#1f6feb22` + border；浅色：`#ddf4ff` + border）。

Logo 固定在 Ribbon 顶部，28×28px 圆角方块。

## 左面板

### 树浏览模式（Browse）

- **连接状态徽章**：面板顶部显示当前连接地址 + 绿色/灰色状态点
- **搜索框**：路径搜索输入框（现有功能）
- **节点树**：递归树结构，支持展开/折叠，点击节点在右侧加载详情
  - 当前选中节点：高亮背景 + 左侧彩色竖条（JSON 绿、二进制红、文本蓝）
  - 有子节点：显示 ▶/▼ 展开箭头
  - 叶子节点：显示 ●/○ 表示有/无数据
- **拖拽手柄**：面板右边缘，鼠标悬停变蓝，拖拽调整宽度

不再显示「收藏路径」和「最近访问」区块（从侧边栏移除，简化左面板）。

### 节点右键菜单

在树面板中，右键任意节点弹出上下文菜单，包含以下操作：

| 操作 | 说明 |
|------|------|
| 创建子节点 | 弹出对话框，输入子路径名称和初始数据（可为空），确认后调用 `create_node` 命令 |
| 删除节点 | 弹出确认对话框（显示完整路径），确认后调用 `delete_node` 命令；若节点有子节点，额外提示「将递归删除所有子节点」 |
| 复制路径 | 将节点完整路径复制到剪贴板，无需对话框 |
| 刷新 | 重新加载该节点的子节点列表 |

**菜单行为：**
- 右键点击时，同时选中该节点（高亮显示）
- 菜单在点击位置弹出，点击菜单外区域或按 Esc 关闭
- 创建/删除操作成功后自动刷新树面板对应位置
- 操作失败（权限不足、节点已存在等）在菜单区域下方或编辑区显示内联错误

**组件：** 新建 `src/components/tree-context-menu.tsx`，通过 `position: fixed` 定位，接收触发节点路径和屏幕坐标作为 props。

### 连接管理模式（Connections）

- **顶部工具栏**：标题「连接管理」+ 「+ 新建」按钮
- **连接卡片列表**：每张卡片显示连接名称、地址、连接状态（绿点=已连接，灰点=未连接）
  - 点击卡片：右侧显示该连接的详情/编辑表单
  - 选中状态：高亮边框

## 右侧内容区

### 树浏览模式下

**（1）节点路径 header**
- 显示当前节点完整路径
- 数据类型标签（JSON · 可编辑 / 文本 · 可编辑 / 格式数据 · 只读 / 二进制 · 只读）
- 右侧：未保存标记（● 未保存，红色，仅在有 draft 时显示）

**（2）Stat 元数据网格**
- 4 列网格，展示完整 ZK Stat 字段：
  - `dataVersion`、`cVersion`、`aclVersion`、`numChildren`
  - `dataLength`、`ephemeral`（是/否）、`mZxid`、`czxid`
  - `mtime`（跨两列）、`ctime`（跨两列）
- 字段名浅灰色，值白色；Zxid 值使用蓝色（hex 格式）
- 背景略深于编辑区，与编辑区形成层次区分

**（3）编辑器工具栏**
- 左侧：RAW / 格式化 切换（格式化为当前视图模式，RAW 为原始字节）
- 右侧：「查看 Diff」「放弃修改」「保存」按钮
  - 保存按钮：可编辑节点为蓝色主色；只读节点为禁用灰色（除非开启强制原始编辑）

**（4）编辑器正文**
- 全宽 textarea / 代码视图
- 保留现有的 draft 管理、forceRaw 切换、Diff 面板逻辑（来自任务 6）

### 连接管理模式下

右侧显示选中连接的详情表单：
- 字段：连接地址、用户名（可选）、密码（可选）、超时时间
- 底部操作：「连接」（主色）、「保存」、「删除连接」（红色边框，靠右对齐）
- 表单验证：连接地址必填，提交时校验格式

## 主题系统

### 切换方式
- 默认跟随系统（`prefers-color-scheme`）
- 设置面板可手动覆盖（持久化到 `localStorage`）

| Token | 值 | 用途 |
|-------|----|------|

| `--bg-subtle` | `#161b22` | 面板背景 |
| `--bg-inset` | `#21262d` | 输入框、代码块背景 |
| `--border` | `#30363d` | 边框 |
| `--border-muted` | `#21262d` | 分隔线 |
| `--text-primary` | `#c9d1d9` | 主要文字 |
| `--text-secondary` | `#8b949e` | 次要文字 |
| `--text-muted` | `#484f58` | 标签名、占位符 |
| `--accent` | `#1f6feb` | 主色（选中、按钮、链接）|
| `--success` | `#3fb950` | 已连接、JSON 可编辑 |
| `--danger` | `#f85149` | 只读标签、删除 |

## 可拖拽面板宽度

- 实现方式：纯 CSS + 鼠标事件（`mousedown` / `mousemove` / `mouseup`），不引入第三方库
- 拖拽手柄：左面板右边缘 4px 宽透明区域，`cursor: col-resize`，悬停时显示 `--accent` 颜色
- 拖拽范围：160px – 400px
- 持久化：`localStorage.setItem('zoocute:sidebar-width', width)`，下次启动时恢复
- React 实现：自定义 hook `usePanelResize(defaultWidth, storageKey)`

## 移除项（当前代码中存在，新设计不再需要）

| 移除内容 | 原因 |
|---------|------|
| 顶部 Topbar 连接表单 | 迁移到连接管理模式的 `connection-pane` |
| 右侧 ContextPanel | Stat 信息整合到右侧内容区上方 |
| WorkbenchTabs 标签页栏 | 不再使用多标签页，选中节点直接在右侧显示 |
| 玻璃态样式（backdrop-filter、渐变背景）| 替换为基于 token 的扁平风格 |
| 收藏路径 / 最近访问区块 | 简化左面板，移除（可后续作为独立功能加回） |
