# Zoocute 用户使用手册

Zoocute 是一个 ZooKeeper 桌面客户端，适合用来浏览节点、查看元数据、编辑文本值、检查操作日志，以及使用解析插件读取节点内容。

英文版用户手册见 [USER_GUIDE.md](../USER_GUIDE.md)。  
开发文档见 [README.md](../README.md) 和 [docs/README.zh-CN.md](README.zh-CN.md)。

## 主要界面

Zoocute 主要有 3 个工作区域：

- `Connections`：新建、保存、测试、连接、断开和删除连接配置
- `Browse`：浏览节点树、搜索已加载节点、查看节点详情、编辑内容、对比变更、管理节点
- `Log`：查看最近的 ZooKeeper 操作记录并进行筛选

## 新建连接

1. 打开 `Connections` 页面。
2. 点击 `+ New`。
3. 填写连接表单：
   - `Connection String`：ZooKeeper 地址，例如 `127.0.0.1:2181`
   - `Name`：可选，连接显示名
   - `Username`：可选
   - `Password`：可选
   - `Timeout (ms)`：超时时间
4. 点击 `Test Connection` 测试连接。
5. 点击 `Save` 保存配置。
6. 在连接卡片上点击连接按钮，建立会话。

## 浏览节点

建立连接后：

1. 打开 `Browse` 页面。
2. 在左侧展开节点树。
3. 点击节点查看详情。
4. 查看版本号、时间戳、子节点数量、数据大小等元信息。

关于搜索：

- 在树上方的搜索框中输入关键字。
- 搜索基于当前会话已经加载进索引的节点。
- 应用会在后台继续构建更完整的索引，因此随着连接持续时间增加，搜索结果可能会越来越全。

## 编辑节点内容

对于可编辑节点：

1. 在节点树中打开目标节点。
2. 进入编辑模式。
3. 修改节点内容。
4. 保存到 ZooKeeper。

注意：

- 二进制内容或不可编辑内容可能不支持直接修改。
- 如果你在有未保存草稿时切换节点，应用可能会提示你是否丢弃改动。

## 与服务端值做对比

在保存前如果想确认当前改动：

1. 打开目标节点。
2. 在工具栏中选择差异对比操作。
3. 应用会重新获取服务端最新值，并与你本地草稿进行比较。

这个功能适合用来确认编辑期间服务端是否已经发生变化。

## 创建和删除节点

Zoocute 支持基础的树结构维护：

- 创建节点
- 删除节点
- 某些删除场景支持递归删除

在共享环境或生产环境执行写操作时请务必谨慎。

## 使用解析插件

解析插件可以把原始节点字节流转换成更适合业务阅读的文本结果。

常见使用流程：

1. 打开一个节点。
2. 在工具栏中选择解析插件。
3. 执行解析。
4. 在插件视图中查看输出结果。

如果插件执行失败：

- 应用会展示插件错误信息
- 你可以切回原始视图
- 清单文件无效的插件可能会被自动忽略

## 查看操作日志

打开 `Log` 页面可以查看最近的 ZooKeeper 操作记录。

你可以：

- 刷新日志
- 按成功或失败筛选
- 按连接 ID 筛选
- 清空本地日志列表

日志页适合用来确认连接、读取、保存、创建、删除或插件相关操作是否成功。

## 常见问题

### 连接失败

- 检查 ZooKeeper 地址是否可达
- 确认集群服务是否正常运行
- 如果集群需要认证，检查用户名和密码
- 如果提示超时，可以适当调大超时时间后重新测试

### 搜索不到节点

- 先等待更多树结构数据加载完成
- 先手动展开相关父节点
- 等当前会话完成更多后台索引后再试一次

### 保存失败

- 确认当前账号有写权限
- 确认节点内容是可编辑文本，而不是二进制数据
- 重新打开节点，并与服务端最新值做一次对比

### 插件没有显示

- 确认插件安装正确，并且包含有效的 `plugin.json`
- 重新打开节点，必要时重启应用以触发插件重新发现

## 使用建议

- 先在非生产环境验证写操作流程
- 删除节点前仔细确认目标路径
- 给连接起清晰的环境名称，例如开发、测试、生产
- 对敏感操作完成后及时查看日志
