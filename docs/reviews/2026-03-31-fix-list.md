## P0

### 1. 缓存快照不要在连接初期覆盖 live tree

状态：已完成

问题：
前端在连接成功后已经通过 `listChildren("/", ...)` 拿到了根节点，但缓存快照仍然会被无条件采用。由于后端缓存 bootstrap 是异步的，连接初期拿到的 snapshot 可能是空的或处于 `resyncing` 状态，导致左侧树短暂空白，甚至被旧快照回退。

涉及文件：
- [src/hooks/use-workbench-state.ts](/d:/Workspace/Personal/wudi/zoocute/src/hooks/use-workbench-state.ts)
- [src-tauri/src/zk_core/live.rs](/d:/Workspace/Personal/wudi/zoocute/src-tauri/src/zk_core/live.rs)
- [src-tauri/src/zk_core/cache.rs](/d:/Workspace/Personal/wudi/zoocute/src-tauri/src/zk_core/cache.rs)

已实施修复：
- 为 `shouldReplaceSnapshot` 增加替换判定逻辑，不再无条件替换。
- 避免空的 `resyncing`/低质量 snapshot 覆盖当前树。
- 收紧连接阶段 listener 注册时机，减少首个 cache 事件丢失窗口。

验收结果：
- 新建连接后左侧树不会再被空 snapshot 刷空。
- 相关前端测试已补齐并通过。

### 2. 连接面板需要正确表示“已连接的多个 session”

状态：已完成

问题：
连接面板当前只用 `activeTabId` 判断哪个连接是在线的。结果是多标签场景下，只有当前标签会显示“已连接”，其他已连接 session 会显示成“未连接”，并且按钮行为变成再次连接，导致会话状态被重建。

涉及文件：
- [src/App.tsx](/d:/Workspace/Personal/wudi/zoocute/src/App.tsx)
- [src/components/connection-pane.tsx](/d:/Workspace/Personal/wudi/zoocute/src/components/connection-pane.tsx)
- [src/hooks/use-session-manager.ts](/d:/Workspace/Personal/wudi/zoocute/src/hooks/use-session-manager.ts)
- [src/hooks/use-workbench-state.ts](/d:/Workspace/Personal/wudi/zoocute/src/hooks/use-workbench-state.ts)

已实施修复：
- 用 `sessions` 的 key 集合作为已连接连接集合。
- 将“当前激活连接”和“已连接连接”从 UI 语义上拆开。
- 非当前标签但已连接的行现在会显示在线，并执行断开而不是重连。

验收结果：
- 多会话场景下，连接列表状态显示正确。
- 新增测试覆盖“非激活但已连接 session 的断开动作”并通过。

### 3. 后端断开连接需要真正释放 ZooKeeper client 和 watcher

状态：已完成

问题：
当前 `disconnect_server` 只是把 adapter 从内存 map 里移除，没有显式停止底层 client 或后台 watcher 任务。这样在 UI 看起来已断开后，底层连接仍可能继续存活，长期会造成 session 泄漏。

涉及文件：
- [src-tauri/src/commands.rs](/d:/Workspace/Personal/wudi/zoocute/src-tauri/src/commands.rs)
- [src-tauri/src/zk_core/live.rs](/d:/Workspace/Personal/wudi/zoocute/src-tauri/src/zk_core/live.rs)

已实施修复：
- 为 `LiveAdapter` 增加 `shutdown()`。
- 为 watcher 增加 shutdown 标记检查，断开后不再继续重注册。
- watcher 对 `Client` 的持有改为弱引用，避免后台任务把 client 强撑住。
- `disconnect_server` 在移除 session 前显式调用 `shutdown()`。

验收结果：
- `cargo test --lib` 通过。
- `cargo check` 通过。

## P1

### 4. 搜索索引刷新时要清理消失节点的整棵子树

状态：已完成

问题：
父节点刷新时，当前实现只删除直接子节点索引，不会清理这些子节点下面已经消失的后代。这样树上已经没了的节点，搜索结果里还可能继续出现。

涉及文件：
- [src/hooks/use-node-search.ts](/d:/Workspace/Personal/wudi/zoocute/src/hooks/use-node-search.ts)
- [src/lib/path-search-index.ts](/d:/Workspace/Personal/wudi/zoocute/src/lib/path-search-index.ts)

已实施修复：
- 增加 `childPaths()`，先计算旧 child 与新 child 的差集。
- 对消失的 child 调用 `removeSubtree()`，不再只删直接 children。
- 新增测试覆盖“父刷新后幽灵孙节点残留”的场景。

验收结果：
- 父节点刷新后，搜索结果不会再残留已删除的后代节点。
- 相关测试已通过。
